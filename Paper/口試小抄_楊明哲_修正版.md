# 第一層｜一分鐘救命速查

## A. 30 秒研究摘要

本研究要處理的不是單純分類，而是高維、小樣本且類別不平衡的 DNA methylation feature selection instability。HECS 先以 repeated random under-sampling 建立平衡資料擾動，再整合 ReliefF、XGBoost 與 L1-SVM 三種互補的特徵排序觀點；單一 run 內以 RRA 與 BH-FDR 建立顯著集合，最後跨 5 個 random-seed runs 取最嚴格的 5-of-5 共識，得到固定 21-CpG panel。其主要貢獻是提高可重現性與穩健性，而不是宣稱分類器本身最複雜。Cross-seed 結果支持高度穩健，但 Nested 分析尚未呈現嚴格數學收斂；內部分類結果很高，但因上游 DMP 候選池存在潛在 supervised selection leakage，應視為描述性一致性，外部 frozen-model transfer 才是較直接的泛化證據。

## B. 四個一定要守住的結論

1. **21-panel 的唯一成員定義**：CpG 必須在 5 個 runs 的 run-level RRA 中都達 `BH-q < 0.05`，即 **5-of-5 consensus**。
2. **Hierarchical-RRA 的角色**：只負責跨 run 的全域排序與輔助解讀，**不重新決定 21-panel 成員**。
3. **兩種驗證不可混用**：Nested-from-25 回答 sets 數增加時的收斂；Independent cross-seed 回答固定 N = 25 時對 random seed 的穩健性。
4. **最安全的整體結論**：結果支持高度 cross-seed robustness 與可重現核心，但尚不足以宣稱嚴格數學收斂、完全無偏的內部泛化，或已具臨床效用。

## D. 最可能被攻擊的問題：一句話先答

### 1. 「AUC 都是 1，是不是 overfitting？」

**先答：** 有樂觀偏差的可能，不能只靠 AUC=1 排除 overfitting；TCGA 腫瘤與正常本身高度可分，加上內部流程存在潛在 selection leakage，所以我把內部結果定位為一致性描述，泛化主要看 frozen external transfer。

### 2. 「你不是說 leakage-free 嗎？」

**先答：** 後段 fold-level `Δβ`、imputation、variance filtering、standardization 與模型訓練確實只 fit 在 training fold；但程式稽核顯示，上游 ChAMP DMP 候選池曾使用該 92 人 set 的完整標籤後才切 holdout，因此存在 supervised selection leakage，我會限縮內部績效的解讀。

### 3. 「所以你的研究失效了嗎？」

**先答：** 不會讓所有結果歸零，但會縮小可主張範圍：內部 AUC 不能當作完全無偏泛化估計；HECS 在既定管線下的跨種子穩健性、共識結構，以及完全未使用外部標籤的 frozen transfer，仍是有效證據。

### 4. 「你證明收斂了嗎？」

**先答：** 尚未證明嚴格數學收斂；Nested Jaccard 並未單調上升，也沒有明確 plateau，但 12 個 CpGs 在四個 N 都保留且平均排名相關為 0.715，支持穩定核心正在形成。

### 5. 「為什麼剛好 21 個？」

**先答：** 21 不是人工指定的 K，而是五個獨立種子 runs 中都達 run-level `BH-q < 0.05` 的最嚴格 5-of-5 共識結果。

### 6. 「5 runs 是 5 個獨立驗證嗎？」

**先答：** 是五條獨立 random-seed resampling trajectories，但仍重用同一 TCGA 樣本池，不能稱為五個獨立 cohorts 或五份獨立臨床證據。

### 7. 「外部驗證真的完全獨立嗎？」

**先答：** 外部 cohort 的標籤未參與 TCGA 模型 fitting 或 threshold selection；Ridge 與 TCGA-derived standardization 固定後直接套用，但部分平台缺少 CpGs，因此使用相對應的 TCGA-trained reduced models，且外部樣本數仍有限。

### 8. 「增生 specificity 只有 0.029，模型是不是沒用？」

**先答：** 這表示 panel 對正常內膜與癌的 transfer 較佳，但不能可靠區分 hyperplasia 與 cancer；結果可能反映增生的癌相關甲基化，也可能受到 cohort、平台、批次與前處理差異影響，不能據此宣稱模型把增生『正確辨識成癌前』。

### 9. 「HECS 的 Jaccard 跟 baselines 公平嗎？」

**先答：** 不是完全相同 estimand：baselines 比較 fixed top-21，而 HECS 的 0.690 比較 variable-size run-level `BH-q < 0.05` sets；因此數值可作輔助比較，但不能宣稱是嚴格一對一的同尺度勝負。

### 10. 「不知道答案時怎麼說？」

**安全句：** 這一點目前資料不能直接回答；我能確認的是＿＿＿，而要區分＿＿＿與＿＿＿，需要再做＿＿＿分析。

# 第二層｜完整口試問答

## 一、研究動機與主要貢獻

### Q1｜你的研究問題到底是什麼？

**15 秒回答：** 高維 DNA methylation 資料常存在「多組預測力相近、但成員不同」的好特徵集合；我的問題是：在類別不平衡與抽樣擾動下，能否找到跨 resampling sets 與 random seeds 都可重現的 CpG 共識，而不只追求單次最高 AUC。

**追問時補充：** TCGA-UCEC 有 436 個腫瘤、46 個正常，特徵數遠大於樣本數。單次 split 或單一模型容易因樣本組成與 inductive bias 改變特徵名單，因此本研究將 feature stability 本身設為主要評估標的。

**避免這樣說：** 不要把研究目的縮成「用三個模型把 AUC 做到 1」。分類在此資料上已接近飽和，無法充分區分 feature selectors。

### Q2｜HECS 的創新或貢獻是什麼？

**15 秒回答：** 貢獻是把 RRUS data perturbation、異質模型排名、run-level RRA/BH-FDR 與跨 run k-of-5 consensus 整合成一個可量化穩健性與重現性的流程。

**追問時補充：** HECS 不宣稱發明 ReliefF、XGBoost、L1-SVM 或 RRA；方法貢獻是將它們放在明確的多層共識架構中，並把 Nested convergence 與 Independent robustness 解耦驗證。

**避免這樣說：** 不要稱為全新統計定理，也不要把分類器表現當唯一創新。

### Q3｜什麼是 Rashomon effect？為什麼跟你有關？

**15 秒回答：** 在高維資料中，可能存在多個預測力相近、但特徵組成不同的良好特徵集合，也就是 multiplicity of good feature sets；HECS 透過反覆擾動與共識來尋找這些集合背後反覆出現的訊號。

**追問時補充：** Rashomon effect 是提出穩定性問題的動機，不是用來替不穩定結果辯解。若結果不穩定，仍須用 Jaccard、Nogueira stability、k-of-5 membership 與 rank correlation 如實量化。

**避免這樣說：** 「反正高維資料都會不穩，所以我的結果不穩很正常。」

### Q4｜為什麼不是只做傳統差異甲基化分析？

**15 秒回答：** 差異檢定回答單一資料配置下的群組差異；本研究額外關心特徵在樣本擾動與模型偏好改變後是否仍被選中，因此需要 resampling-based stability analysis。

**追問時補充：** Welch's t-test baseline 本身也可能穩定，不能因為它是單變量方法就否定。HECS 的重點是在同一研究框架下比較不同策略的可重現性、效果量與外部 transfer。

## 二、HECS 方法與設計

### Q5｜請用一分鐘講完整流程。

**15 秒回答：** 每個 set 保留全部 46 normals，從 436 tumors 中無放回抽 46，形成 92 人平衡 set；在 set 內做 5-fold feature ranking，整合 ReliefF、XGBoost、L1-SVM；單一 run 彙整 25 sets 的 ranked lists，以 RRA 及 `BH-q<0.05` 得 run-level set；最後跨 5 runs 取 5-of-5 得到 21-panel。

**追問時補充：** Stage 1 中 model coverage 主要要求 `cov ≥ 1`，selection frequency 作為 fold-level 穩定性與排序描述，不把舊版固定 frequency threshold 說成目前的硬性成員門檻。Hierarchical-RRA 在 cross-run 層級提供全域優先排序，但不改變 21-panel membership。

### Q6｜RRUS 為什麼保留所有 normals，只抽 tumors？

**15 秒回答：** 正常樣本只有 46 例，全部保留可避免再丟失少數類資訊；對 436 例 tumors 反覆欠採樣，則同時完成 1:1 平衡並產生樣本擾動。

**追問時補充：** set 內 tumor sampling 是無放回，跨 sets 同一 tumor 可再次出現。這延續 repeated majority under-sampling 的概念，但 HECS 不是 EasyEnsemble classifier；RRUS 在此主要用於 feature-selection perturbation。

**避免這樣說：** 不要說「RRUS 證明對所有樣本擾動都穩健」。由於 46 normals 固定，穩健性主要反映 tumor subsampling 與 seed perturbation，可能低估對 normal-cohort composition 的敏感度。

### Q7｜為什麼是 ReliefF、XGBoost 與 L1-SVM？

**15 秒回答：** 三者提供互補的 inductive biases：ReliefF 著重局部鄰域差異，XGBoost 可捕捉非線性閾值與交互結構，L1-SVM 提供全域線性稀疏權重。

**追問時補充：** 選擇目的是增加觀點異質性，不是宣稱三者統計獨立或數學正交。Random Forest、elastic net 或 mutual information 都可成為未來擴充與敏感度分析。

**避免這樣說：** 不要說三模型已涵蓋所有可能的生物交互作用，也不要說沒有其他合理模型。

### Q8｜為什麼不用 Random Forest？

**15 秒回答：** XGBoost 已提供 tree-based nonlinear ranking；為控制計算量與方法複雜度，主分析保留三種代表性偏好。這是研究設計選擇，不是 Random Forest 不適用的證明。

**追問時補充：** 若要證明模型組合的充分性，應做 model-set ablation 或加入 RF 後比較穩定性，而不是只用理論敘述。

### Q9｜Stage 1 的 frequency 與 coverage 怎麼解釋？

**15 秒回答：** frequency 描述特徵在 5 folds 與多模型中的反覆出現程度；coverage 描述有多少不同模型支持。現行主線的 Stage 1 保留以 `cov ≥ 1` 為主，frequency 主要用於描述與排序。

**追問時補充：** Stage 1 是寬鬆候選生成，不是最終高信心宣告。真正嚴格的資格來自單一 run 的 RRA `BH-q<0.05`，以及跨 runs 的 5-of-5 consensus。

### Q10｜RRA 在做什麼？

**15 秒回答：** RRA 檢驗某特徵是否在多份 ranked lists 中，比隨機情況更常出現在前端；再以 BH-FDR 控制多重檢定，建立 run-level significant set。

**追問時補充：** 每個 run 內有 25 個 set-level ranked lists。RRA 把不同列表長度與排序整合成顯著性；`BH-q<0.05` 是 run-level membership 的門檻。

### Q11｜為什麼用 BH-q < 0.05？

**15 秒回答：** 因為同時評估大量 CpGs，未校正 p 值會累積 false discoveries；BH 控制的是預期 false discovery rate，0.05 是預先固定且常見的錯誤率標準。

**避免這樣說：** `q<0.05` 不等於每個位點有 95% 機率為真，也不證明生物因果。

### Q12｜為什麼最後是 21 個，而不是 10、20 或 30？

**15 秒回答：** 21 是資料驅動的最嚴格 cross-run consensus size：五個 runs 中皆達 run-level `BH-q<0.05` 的 CpGs 共 21 個，並非事後指定 top-K。

**追問時補充：** k-of-5 ladder 是 5/5=21、4/5=23、3/5=27、2/5=31、1/5=44，顯示放寬跨 run 重現要求後集合逐步擴大。

### Q13｜Hierarchical-RRA 會不會又選一次特徵？

**15 秒回答：** 不會。21-panel 成員資格已由 5-of-5 run-level `BH-q<0.05` 固定；Hierarchical-RRA 只對 cross-run candidates 提供整體優先順序與輔助解讀。

**避免這樣說：** 不要說 21 個是由 hierarchical q-value 的虛線切出來的。

### Q14｜為什麼是 25 sets、5 runs？

**15 秒回答：** 這是兼顧抽樣覆蓋、計算成本與跨種子評估的預先固定主分析規模；N=25 是 robustness stress test 的固定點，不是已被證明的全域最佳值。

**追問時補充：** Nested N=10、15、20、25 用來檢查資訊增益與收斂趨勢。現有結果沒有清楚 plateau，因此不能用數據宣稱 25 已經足夠或最優；更大的 N 與更多 runs 是合理未來工作。

## 三、Nested convergence 與 cross-seed robustness

### Q15｜Nested 與 Independent 設計有何不同？

**15 秒回答：** Nested 固定同一 seed trajectory，依序取前 10、15、20、25 sets，只改變 sets 數量，所以回答 convergence；Independent 固定 N=25、改變 random seeds，所以回答 seed robustness。

**避免這樣說：** 不要用 Independent results 證明 N 已收斂，也不要用 Nested results 宣稱跨 seed 穩健。

### Q16｜你的結果真的收斂嗎？

**15 秒回答：** 尚未達到可宣稱的嚴格數學收斂。panel sizes 為 14、18、19、28，Jaccard 介於 0.500–0.684 且未單調增加，因此沒有清楚 plateau。

**追問時補充：** 正向證據是四個 N 共有 12 個 CpGs，且 mean Spearman 為 0.715，顯示核心 membership 與主要 ranking 具一定一致性。最精確結論是「出現穩定核心，但整體集合尚未完全收斂」。

**避免這樣說：** 不要把某一對 N 的較高 Jaccard 當成整段 trajectory 已收斂。

### Q17｜為什麼 N 增加，顯著集合反而變大？

**15 秒回答：** 較多 ranked lists 可能使反覆出現、但先前證據不足的特徵取得更穩定的聚合證據，因此偵測敏感度增加；這與「集合必須縮小才叫收斂」不是同一件事。

**追問時補充：** 但不能說 N 增加必然使 p 值單調下降或 panel 單調擴大。RRA 結果取決於排名分布、候選出現模式與 BH 校正；實際軌跡仍應以數據呈現。

### Q18｜Independent cross-seed 的主要證據是什麼？

**15 秒回答：** 五個 run-level sets 的 mean Jaccard 為 0.690，Nogueira stability 為 0.814（95% jackknife interval 0.776–0.853），排序 mean Spearman 為 0.793，並有 21 個 CpGs 在 5/5 runs 顯著。

**追問時補充：** 這些結果共同支持對 random-seed perturbation 的高度穩健性，但仍是同一 TCGA 樣本池內的演算法重現性，不等同跨族群或跨醫院重現性。

### Q19｜Jaccard 0.690 與 Nogueira 0.814 為何不同？

**15 秒回答：** Jaccard 是兩兩集合交集除以聯集；Nogueira stability 則在共同特徵空間中依選取頻率做 chance correction。定義與尺度不同，不能直接要求兩個數字相同。

**追問時補充：** 本研究 Nogueira 的 `d=8,031` 是五個 runs 候選聯集，因此 0.814 是在這個條件式 universe 下的穩定性，不是所有 450K probes 上的無條件值。

### Q20｜只有 5 runs，Nogueira 的 CI 可靠嗎？

**15 秒回答：** 它提供現有五個 runs 下的 jackknife 不確定性描述，但 run 數少，區間本身仍有限；應視為支持性證據，而不是精確到小數點後三位的母體保證。

**追問時補充：** 更理想作法是增加 runs，並檢查不同候選 universe、不同 set sizes 與不同 normal resampling schemes 下的敏感度。

### Q21｜Spearman 對缺失特徵補 max+1 合理嗎？

**15 秒回答：** 它把「未出現在某 run」視為低於該 run 已觀察排名的右設限，讓共同 scope 可計算相關；是務實的 padding rule，但不是唯一或保證保守的作法。

**追問時補充：** 多個缺失特徵被賦予相同尾端名次，可能提高 ties 並影響 rho。因此 Spearman 主要描述 shared candidate scope 內的排序一致性，membership robustness 應優先看 Jaccard、k-of-5 與 Nogueira stability。

## 四、資料洩漏、AUC 與內部驗證

### Q22｜內部 AUC=1.000 是不是 overfitting？

**15 秒回答：** 不能只由 AUC=1 判定或排除 overfitting。TCGA 腫瘤與正常在這些 CpGs 上高度可分，但上游候選生成與重複樣本使用可能使內部估計偏樂觀，所以我只把它解讀為既定流程下的分類一致性。

**追問時補充：** 125/125 sets 的 AUC 都是 1，specificity 都是 1，mean sensitivity 0.9928；這顯示 ranking separation 強，但不是無偏 generalization proof。外部 GSE155760 的 AUC 0.977 更接近真正 transfer evidence。

**避免這樣說：** 「125 次都是 1，所以不可能 overfit。」重複評估若共享樣本與上游特徵資訊，不等於 125 份獨立證據。

### Q23｜流程到底有沒有 leakage？

**15 秒回答：** 有一個需要誠實揭露的風險：後段 fold-level preprocessing 是 train-only，但上游 `champ.DMP()` 在 holdout split 前使用整個 92 人 set 的 labels 篩出 DMP probes，後續模型輸入再與這份 DMP list 取交集，因此構成潛在 supervised selection leakage。

**追問時補充：** 程式流程可分兩段：

1. 上游 ChAMP：用完整 set 的 phenotype 計算 DMP，並只輸出達 adjusted p-value 門檻的 probes。
2. 下游 HECS：建立 holdout 後，fold-train-only 計算 `Δβ`、imputation、variance filtering、standardization 與模型 ranking。

第二段的實作是正確的，但不能抵消第一段已看過完整 labels 的影響。

**避免這樣說：** 不要回答「因為 StandardScaler 只 fit training fold，所以整體完全 leakage-free」。

### Q24｜這個 leakage 會讓整篇論文無效嗎？

**15 秒回答：** 它會削弱內部分類績效的因果解讀與無偏性，但不會讓所有結果自動歸零。較安全的結論是：穩定性結果描述既定候選生成管線下的重現性；內部 AUC 只作一致性描述；外部 frozen transfer 因未使用外部 labels，仍提供獨立於外部 outcome 的支持。

**追問時補充：** 即使 panel 最初用完整 TCGA labels 選出，只要外部 cohort 完全未參與選模與 fitting，這是正常的 training-to-external-test 關係。真正需要修正的是把 TCGA 內部 holdout 說成完全未被任何 supervised upstream information 影響。

### Q25｜怎麼徹底修正 leakage？

**15 秒回答：** 將 ChAMP DMP／supervised candidate filtering 也移入每個 outer-training split，所有 probe selection、imputation、scaling、RRA parameter choice 都只在 outer training 完成，最後只評估一次 untouched outer test 或 external cohort。

**追問時補充：** 若計算成本允許，使用完整 nested CV；若樣本有重複或同一病人多 specimen，則用 group-aware split。最終 panel 可在方法選定後以全 TCGA 重訓，再鎖定模型做真正外部驗證。

### Q26｜有 tumor 誤判，為什麼 AUC 還是 1？

**15 秒回答：** AUC 評估正負樣本的整體排序，不固定在 0.5 threshold；若所有 tumors 的 score 仍高於所有 normals，即使最低 tumor score 低於分類 threshold 而被誤判，AUC 仍可為 1。

**追問時補充：** 因此 AUC=1 與 sensitivity<1 可以同時成立。前者是 ranking metric，後者是某一 decision threshold 下的分類率。

### Q27｜125 個 holdouts 算 125 次獨立測試嗎？

**15 秒回答：** 不算。跨 sets 同一 tumor 可重複、46 normals 固定，且 final panel 是跨 runs 共識；因此 125 個結果是 repeated internal evaluations，不是 125 個獨立測試集。

**追問時補充：** 可報告分布、最小值與一致性，但置信區間若假設 125 次完全獨立會過度樂觀。真正獨立性要依 patient/cohort level 判定。

### Q28｜那內部結果還能說什麼？

**15 秒回答：** 可以說 21-panel 在不同 tumor subsamples 下的 Ridge ranking 與 threshold performance 非常一致；不能說它提供完全無偏、可直接外推臨床族群的性能估計。

## 五、Baseline Ladder 與公平比較

### Q29｜為什麼做 Baseline Ladder？

**15 秒回答：** 用 t-test、XGB-only、Ensemble 與完整 HECS 逐步增加方法成分，觀察模型異質性與跨 set 聚合對穩定性、效果量與外部 transfer 的影響。

**追問時補充：** XGB-only → Ensemble 比較較接近加入模型異質性的 ablation；Ensemble → HECS 同時涉及跨 set aggregation、顯著集合與跨 run consensus，不能稱為完全單一變因。

### Q30｜四種方法的穩定性數字是多少？

| 方法 | mean Jaccard | Nogueira stability [95% interval] |
|---|---:|---:|
| t-test | 0.316 | 0.475 [0.391, 0.559] |
| XGB-only | 0.231 | 0.370 [0.252, 0.487] |
| Ensemble | 0.493 | 0.656 [0.584, 0.729] |
| HECS | 0.690 | 0.814 [0.776, 0.853] |

**安全解讀：** 在目前分析定義下，加入模型異質性後 stability 上升，完整 HECS 最高；但 HECS 的 Jaccard 使用 variable-size `BH-q<0.05` sets，而 baselines 主要是 fixed top-21，所以不是完全一致的集合估計量。

### Q31｜t-test 的 Jaccard 反而比 XGB 高，怎麼解釋？

**15 秒回答：** 這是真實結果，代表在本研究的擾動設定下，t-test top-21 比 XGB top-21 更穩定；不能因 t-test 是 deterministic 或單變量方法就把它排除。

**追問時補充：** Deterministic 演算法在輸入樣本改變時仍可能改變排名。其穩定性 0.316／0.475 並非零，只是低於 Ensemble 與 HECS。預測能力、效果量、穩定性與生物可解釋性是不同面向。

### Q32｜HECS 的 Jaccard 跟前三法算法不同，還能比較嗎？

**15 秒回答：** 可作方向性、輔助比較，但不應稱為完全同尺度。Jaccard 對 fixed-size 與 variable-size sets 的行為不同，HECS 還經過 BH threshold 與 consensus aggregation。

**追問時補充：** Nogueira analysis 讓四種方法共用 `d=8,031` 的 reference space，提高部分可比性；但選取規則仍不同，因此結論應是「HECS 在目前完整流程下呈現較高穩定性」，而不是單一元件的純因果效果。

### Q33｜分類都飽和，Baseline Ladder 還有意義嗎？

**15 秒回答：** 有。TCGA 內部各方法 Ridge 表現都接近或達 1，正好說明 accuracy/AUC 無法區分 feature selectors；穩定性、外部 transfer 與效果量才提供額外辨識力。

### Q34｜外部 AUC 只高一點，有實質意義嗎？

**15 秒回答：** GSE155760 上 t-test、XGB-only、Ensemble 與 HECS AUC 約為 0.965、0.953、0.959、0.977，差距不大且樣本只有 36，因此不宣稱顯著優越；較合理的貢獻仍是穩定性與可重現性。

**追問時補充：** HECS 在該 cohort 有 specificity 1.000、sensitivity 0.913。若要比較 AUC 差異是否具統計意義，需要預先指定的 paired ROC test／bootstrap CI，且樣本量限制仍在。

### Q35｜下游分類器為什麼統一用 Ridge？

**15 秒回答：** Ridge 提供固定、相對簡單的線性分類器，使 Baseline Ladder 主要比較上游特徵集合，而不是不同 classifier capacity。21-panel 主分析與 baseline comparison 都使用 Ridge，但必須分清各自的資料切分與結果表。

**避免這樣說：** 不要把 125-set 內部結果、單次全 TCGA baseline holdout 與外部 transfer 的 AUC 混成同一實驗。

### Q36｜參數是不是事後調出來的？

**15 秒回答：** 主分析使用固定設定，例如 fold-train `|Δβ|≥0.2`、variance threshold `1e-6`、standardization std floor `1e-2`、模型組合與 BH-q 門檻；它們有方法學或數值穩定性理由，但不宣稱是唯一最佳值。

**追問時補充：** `|Δβ|=0.2` 是甲基化陣列研究中常見的實務門檻，而非普遍生物真理；`std floor=1e-2` 是避免低變異特徵標準化後爆大的防護。完整敏感度分析仍有限，應列為未來工作。

## 六、外部固定模型驗證

### Q37｜外部驗證的模型到底怎麼訓練？

**15 秒回答：** Ridge 在 TCGA-UCEC 訓練後固定，外部 cohort 不重新 fitting，也不使用外部 labels 選參數；外部資料以 TCGA-derived 均值與標準差轉換後直接預測，且未做跨 cohort ComBat。

**追問時補充：** 若外部平台缺少 panel CpGs，則用該 cohort 可用 CpGs 在 TCGA 上訓練對應 reduced model，再固定轉移。例如 GSE155760 使用 reduced-18；這維持 external labels 不進入 fitting，但不同 cohort 的 feature subset 不完全相同。

### Q38｜外部 cohort 結果總表怎麼背？

| Cohort | 組成 | Model | 結果 | 必要註解 |
|---|---|---|---|---|
| GSE155760 | 13 normal、23 tumor | reduced-18 | AUC 0.977；sens 0.913；spec 1.000 | 主要兩類 external transfer；n=36 |
| GSE73949 | 17 normal specimens | full-21 | specificity 1.000 | 單類；不能算 AUC |
| GSE90060 | 34 normal biopsies | full-21 | specificity 1.000 | 來自 17 位女性的 paired biopsies，不是 34 位獨立受試者 |
| GSE178610 | 80 tumors | reduced-17 | sensitivity 0.950 | 49 fresh-frozen + 31 FFPE；樣本處理具異質性 |
| GSE93589 | 9 tumors | reduced-20 | sensitivity 1.000 | 樣本非常少 |
| GSE67116 | 8 hyperplasia、86 tumors | full-21 | AUC 0.820；sens 0.953；spec 0.375 | 對照不是正常內膜 |
| GSE136791 | 34 hyperplasia、69 tumors | reduced-20 | AUC 0.627；sens 1.000；spec 0.029 | 對照不是正常內膜 |

### Q39｜為什麼單類 cohort 不合併算 AUC？

**15 秒回答：** AUC 需要同一可比較資料來源中的正、負兩類。若把不同研究的全 normal 與全 tumor 硬合併，AUC 可能同時捕捉 cohort、平台、批次與國家差異，而不是癌症差異。

**追問時補充：** 因此 GSE73949／GSE90060 只報 specificity；GSE178610／GSE93589 只報 sensitivity。這是較保守且可解釋的報告方式。

### Q40｜GSE155760 為什麼只用 18 個 CpGs？

**15 秒回答：** 因平台映射與資料可用性，21-panel 中只有 18 個可可靠對齊，因此使用 TCGA 上對應訓練的 reduced-18 Ridge，再 frozen transfer 到 GSE155760。

**追問時補充：** 這不是用外部 outcome 挑 18 個；但 reduced-18 與 full-21 不是完全同一模型，所以外部結果證明的是可映射子面板的 transferability。

### Q41｜GSE136791 specificity 0.029 怎麼解釋？

**15 秒回答：** 這表示 34 個 hyperplasia 中只有 1 個在既定 threshold 下被判為非癌，panel 對 hyperplasia 與 cancer 的臨床鑑別能力很弱。

**追問時補充：** 可能原因包括 hyperplasia 與 carcinoma 的甲基化連續性，也包括 platform、batch、組織組成、地區、前處理與 reduced-model 差異。現有設計無法把這些原因分開，因此只能說「與多種生物及技術解釋一致」。

**避免這樣說：** 不要把低 specificity 直接包裝成模型成功偵測癌前病變。

### Q42｜外部結果能證明臨床效用嗎？

**15 秒回答：** 不能。它支持跨 cohort transfer 的可行性，但距離臨床效用仍缺少前瞻性、多中心、連續病程、calibration、decision-curve 與預先鎖定 cutoff 的驗證。

## 七、圖表被指著問時

### Q43｜Cross-seed heatmap 要怎麼講？

**15 秒回答：** 熱圖顯示五個 independent seed runs 的 run-level results 具有高度相似結構；定量證據是 mean Jaccard 0.690、mean Spearman 0.793 與 Nogueira 0.814。

**追問時補充：** 若圖只顯示 hierarchical top-25 shared rows，它是經最終排名選出的描述性視圖，不是獨立驗證，也不能單靠視覺相似宣稱穩健。

### Q44｜`hierarchical_rra_volcano.png` 是 volcano plot 嗎？

**15 秒回答：** 嚴格來說不是典型 volcano plot。橫軸是 hierarchical rank 的對數，縱軸是 `-log10(hierarchical BH-q)`，兩軸都來自同一 ranking/significance procedure；較準確稱為 **Hierarchical-RRA significance landscape**。

**追問時補充：** 圖中顯著虛線只表示 hierarchical q threshold；21-panel 的資格仍來自 5/5 run-level BH-q membership。即使 hierarchical q<0.05 的候選很多，也不代表都進入 21-panel。

### Q45｜21-panel beta heatmap 能證明分類效能嗎？

**15 秒回答：** 它顯示 TCGA 腫瘤與正常在 21 CpGs 上有清楚的甲基化模式差異，與 mean `|Δβ|=0.552` 相符；但 heatmap 使用已選定特徵，只是描述性證據，不能取代獨立性能評估。

### Q46｜Baseline Ladder 圖怎麼講？

**15 秒回答：** TCGA 內部 AUC 接近飽和，方法差異主要出現在 stability；XGB-only 到 Ensemble 的 Jaccard／Nogueira 明顯上升，完整 HECS 在目前流程下最高，但 HECS 與 baselines 的 set definition 不完全相同。

## 八、生物學解讀

### Q47｜21 個 CpGs 有沒有生物意義？

**15 秒回答：** 21 個位點在 TCGA 中有大型群組差異，平均 `|Δβ|=0.552`、平均 `|d|=3.830`；其中 11 個 CpGs 可對應 9 個已知 genes，其餘 10 個未直接對應已知 gene。部分位點具有既有子宮內膜癌或甲基化標記文獻線索，但整體仍屬 biomarker candidates。

**追問時補充：** 較常被問到的包括 TRIM15、CYTH2、SIM1、TBX15 與 CACNA1H。文獻支持程度不一，應逐位點區分 direct evidence、間接 plausibility 與僅有 genomic context。

**避免這樣說：** 不要由關聯與效果量推論這些基因驅動癌症，也不要把未註釋 CpGs 稱為沒有生物意義。

### Q48｜效果量很大，是否表示可臨床診斷？

**15 秒回答：** 大 `Δβ` 與 Cohen's d 顯示 TCGA 腫瘤／正常差異明顯，有利於 assay development；但臨床診斷還需跨族群可重現性、技術精密度、cutoff calibration、病灶譜、前瞻性效能與臨床淨效益。

## 九、限制與未來工作

### Q49｜你認為最重要的限制是什麼？

**15 秒回答：** 第一是上游 DMP 候選池的潛在 supervised selection leakage，使內部績效可能偏樂觀；第二是 robustness 仍主要在同一 TCGA 樣本池內，且 normals 固定；第三是外部 cohorts 樣本數、平台、組織與可用 probes 不一致。

**追問時補充：** 其他限制包括只有 5 runs、Nested 尚無 convergence plateau、參數敏感度分析有限、不同 stability estimands 的可比性，以及尚未有功能或前瞻性臨床驗證。

### Q50｜如果再做一次，優先改什麼？

1. 將 DMP／所有 supervised candidate filtering 完整移入 outer-training folds。
2. 建立真正 untouched external test，或先鎖定 pipeline 再做多中心驗證。
3. 增加 random-seed runs 與 N>25，檢查 stability plateau 及 CI。
4. 同時擾動 normal 與 tumor，評估結果對少數類組成的敏感度。
5. 系統性比較 `Δβ`、variance threshold、Stage 1 與 BH-q 設定。
6. 加入 Random Forest／elastic net 等 model-set ablation。
7. 對 reduced panels、平台映射、批次效應與組織來源做分層分析。
8. 進行 targeted methylation assay、功能驗證、calibration 與 decision-curve analysis。

### Q51｜你的研究一句話結論？

**15 秒回答：** HECS 在 TCGA-UCEC 的既定候選生成流程下，萃取出一個對 random-seed perturbation 高度可重現的 21-CpG consensus panel，並在 frozen external transfer 中保留良好區辨力；但 Nested 尚未證明嚴格收斂，且內部分類結果須因上游 selection leakage 而保守解讀。

| Run-level significant set | 單一 run 內經 RRA 後達 `BH-q<0.05` 的 CpGs                 |
| k-of-5 consensus          | 某 CpG 在至少 k 個 runs 的 run-level significant sets 中出現 |
| 21-panel                  | 最嚴格 k=5，也就是 5/5 共識集合                                |
| Hierarchical-RRA          | 跨 run 全域排序；不決定 21-panel membership                  |
| Nested convergence        | 同一 trajectory 的 N=10/15/20/25 前綴比較                  |
| Independent robustness    | 固定 N=25，比較不同 random seeds                           |
| Stability                 | 特徵在資料或 seed 擾動後仍被選中的程度                              |
| Generalization            | 模型對真正未見資料的預測能力；不等同 stability                        |
| AUC                       | threshold-independent ranking discrimination        |
| Sensitivity／specificity   | 固定 threshold 下對正類／負類的辨識率                            |

# 附錄 C｜20 秒收尾稿

本研究最重要的結果不是 TCGA 內部 AUC 達到 1，而是透過明確的擾動與共識架構，找到跨五個 random-seed runs 皆重現的 21-CpG signature。現有結果支持高度 seed robustness 與初步 external transferability；同時，我也會保守承認 Nested 尚未出現嚴格 convergence plateau、上游 DMP 預篩可能使內部績效偏樂觀，以及外部 cohorts 仍有限。下一步應以完整 nested pipeline 與預先鎖定的多中心外部驗證確認其泛化與臨床價值。
