# 口試問答小抄 — HECS / TCGA-UCEC DNA 甲基化

> 定稿版：下游分類器是 **Ridge**、已含 **7 個 GEO 外部 cohort**。

---

## ★ 三個無論如何守住的底線

1. **內部 AUC 1.0 = 資料飽和**（正常與腫瘤本就高度可分），不是方法優點、也不是穩定性指標。穩定性歸 **Nogueira 0.814**，泛化歸 **外部 0.977**。
2. **21 個位點由 k-of-5 共識門檻決定**（5 輪次全達 BH-q<0.05），**不是「收斂」**、不是人為指定。
3. **21 位點是「候選起點」**，不是子宮內膜癌的因果／臨床標記。生物證據僅到註釋層級。

# 一、核心主軸題

### 1｜內部 AUC=1.000 是不是 overfitting？
- 不是過擬合，也**不是我的穩定性指標**——1.0 反映**任務飽和**：正常與腫瘤甲基化本就高度可分。
- 內部這層證明的是「表現不隨 125 種樣本組成飄動」＝**表現一致性**；真正泛化證據是**外部 0.977**，穩定性指標是 **Nogueira 0.814**。

### 2｜內部 AUC=1.0 是不是資料洩漏？
- 不是。所有前處理（離群值、補值、變異門檻、z-score）**只用該抽樣集合的訓練子集估計**，再套到測試子集。
- ChAMP QC/探針過濾**不用類別標籤**（無監督）；BMIQ 是**逐樣本**校正；DMP 統計量**未被用來預先篩選候選探針池**。
- 正因內部會飽和，我才另做外部 GEO 驗證檢驗真泛化。

### 3｜分類都飽和，你的貢獻到底是什麼？
分類飽和、四法 AUC 相近 → **分類效能無法區分方法**；能區分的是**特徵篩選穩定性**，這就是我的貢獻。整篇定位在**可重現性**，非分類效能。

### 4｜為什麼是 21 個位點？
21 **不是我指定、也不是集合大小收斂**（集合大小隨 N 遞增是檢定力效應）。它由**跨種子 k-of-5 共識門檻**決定：5 個獨立種子輪次**全部**達 BH-q<0.05 者剛好 21 個。放寬 4/5=23、3/5=27。
- ⚠️ 絕不用「自然收斂」——軌跡集合大小 14→18→19→28 沒有平台期。

### 5｜內部有沒有分錯的？
有。少數集合中 10 個腫瘤測試樣本會出現 1 個誤判（敏感度最低 0.900）；但兩類預測機率**排序仍可完全分開**，故 AUC 維持 1.000。

# 二、方法設計題（why 系列）

### 6｜為什麼要做階梯式基準比較？
它是**受控的消融實驗**，四個理由：
1. HECS 疊了多層，一個總數字看不出**哪一層在起作用** → 逐層加回、分離邊際貢獻。
2. 分類飽和無鑑別力 → 需換一條**沒飽和的軸**（穩定性、效果量）來比。
3. **四法都輸出 21 特徵、都用同一 Ridge** → 固定面板大小與分類器，差異只歸因於篩選策略。
4. 直接回答「優勢來自完整框架，還是單一元件？」
- 三組相鄰比較：t-test→XGB（統計 vs 單模型）／XGB→Ensemble（**異質集成，穩定性最大跳升 0.370→0.656**）／Ensemble→HECS（**不提升 AUC，而是固定成單一可重現輸出 0.656→0.814**）。
- 加分：t-test 跨種子 Jaccard(0.316)反而高於 XGB(0.231)，但只反映單變量統計量本來就確定，非對抽樣更穩健。

### 7｜為什麼是這三種模型？不用別的？（專題）
**口徑總則**：不要說「這三個效果最好」。強調**選擇原則是異質性與互補性**，且三模型是**特徵排序工具、不是最終分類器**（下游 Ridge），別跟分類效能混談。

**完整版**：選 ReliefF／XGBoost／L1-SVM，是因為 HECS 目的是**從不同建模假設中找可重現的共同訊號**，不依賴單一模型偏好。
- ReliefF＝局部鄰域與樣本拓撲；XGBoost＝非線性閾值與特徵交互；L1-SVM＝全域線性 margin-based 稀疏 → 三種**互補視角**（互補指假設與評分機制不同，非統計完全正交）。
- 一個 CpG 若能在**不同模型、不同重抽樣、不同 seed** 下反覆出現，比只被單一演算法選中更該視為 stable feature。

**一句話版**：不是找單一最好演算法，而是**刻意選三種不同 inductive bias 的評分方法**，讓結果由**跨模型／跨 resampling／跨 seed 的重現性**決定，而非單一模型偏好決定。

**常見追問**
- **不用 Random Forest？** RF 與 XGBoost 同屬**樹家族**，一起放會讓樹視角被重複加權；故選一個代表性 boosting tree，再搭鄰域與稀疏線性維持平衡。
- **為什麼剛好三個？** 關鍵是**加新視角、非加重複演算法**。三個已涵蓋局部／非線性樹／全域稀疏線性三大面向，同時控制 resampling×5-fold×多 runs 的計算成本。
- **能證明換模型也一樣？**（保守）只支持**三模型 HECS 架構內**的重現性，不宜宣稱對所有演算法都不變；要答 model-set dependence 需做 ablation／replacement sensitivity（如以 RF 或 ElasticNet 替換其一，檢查面板 overlap）。

### 8｜RRUS 為什麼只擾動腫瘤、正常樣本全保留？
- 正常僅 46 例（少數類），全保留是為了**不再稀釋稀少的少數類資訊**（EasyEnsemble 概念，Liu 2009）；擾動集中在多數類腫瘤端引入抽樣變異。
- ⚠️ 誠實承認這使穩定性估計**偏樂觀**，但不失效：跨種子時腫瘤組合不同仍造成 |S_r| 差異（28/28/27/30/33、Jaccard 低到 0.605）→ 擾動確實在影響結果；且**外部特異度 1.000** 提供獨立於固定正常組的佐證。未來以更多正常 cohort 交叉檢驗。

### 9｜為什麼是 25 sets / 5 runs？
- **25 sets**＝軌跡收斂分析的**預設最大抽樣深度**（非收斂點），也是主分析設定。
- **5 runs**＝跨種子穩健性的**獨立壓力測試**。
- 兩者功能不同、不是同一個參數問題。不宣稱是唯一最佳，是主分析設定。

### 10｜集合大小為何隨 N 增加？合理嗎？
- 合理，是**檢定力效應**：N 越大、每個特徵在 RRA 累積的證據越多、p 值越小 → 原本訊號偏弱的邊際位點才通過 FDR。跟樣本數變大、檢定力上升同理。
- 穩定的是**核心與排序**：k-of-4 有 **12 個核心零脫落**、跨深度 Spearman 0.715。集合是往外長、不是換血。
- ⚠️ 這題**別講「收斂」**形容集合大小；收斂只收在核心特徵與排序上。

### 11｜3.4 中未入選特徵的名次為何補「max+1」？（Spearman 補值）
- **用途**：這是算 **Spearman 排序相關**時的補值，在兩份顯著名單的**聯集**上計算；某特徵若在其中一份沒出現就沒有名次，必須補一個。
- **規則**：補成該清單**最大名次+1、並列最後**（排在所有實際入選特徵之後）。
- **為什麼可以這樣做**：
  1. **忠實**：沒出現＝在那份裡沒被排進前段，放最後正是「未被優先」最忠實的編碼。
  2. **保守、不灌水**：高名 A、補到最後 B 會**拉低** ρ → 懲罰不一致，不會高估穩定性。
  3. **拒絕只取交集**：交集會丟掉「只出現一邊」的特徵，而那正是要量測的不穩定 → 聯集+補最後才保留不一致訊號。
  4. **與 RRA 一致**：3.3.5 RRA 也把缺席特徵補為最差標準化排名（=1）；max+1 是同原則的離散版。
- **追問「tie 太多會扭曲嗎？」**：Spearman 對並列用**平均秩**處理（標準做法），方向上保守（降低而非拉高相關）；且另有 Jaccard、Nogueira 交叉佐證，不單靠 Spearman。

### 12｜前處理／排序參數是不是太主觀、事後調的？
所有參數（Δβ=0.2、top-100、VarianceThreshold 1e-6、STD floor 1e-2 等）**主分析前就固定，非 post-hoc 調參**。Δβ=0.2 是文獻常用差異幅度門檻（Fennell 2022）。未來工作已列出要做門檻敏感度分析。

### 13｜樣本會跨集合重複，內部 125 集還算獨立測試嗎？
不把內部 125 集稱為獨立測試——它是**固定共識集合在不同抽樣背景下的重複內部驗證**，看表現會不會飄動。真正的獨立性由**外部 GEO cohort** 提供（外部樣本從未參與篩選、標準化估計或訓練）。內部檢查「穩定」、外部檢查「泛化」。

# 三、結果與外部驗證解讀題

### 14｜增生對照特異度低（GSE136791 只有 0.029），模型是不是沒用？
- **先擋批次質疑（跨平台隔離）**：同為 EPIC、同樣沒做 BMIQ 的 GSE155760（正常對照）特異度仍 1.000、GSE178610 敏感度 0.950 → 變數是「對照是正常還是增生」，**不是平台**。
- 兩個增生 cohort（英國 450K／日本 EPIC）不同國家不同平台卻**同向下降**，不像隨機批次。
- 與「增生屬癌前病變、甲基化部分趨近癌症」一致；**但不排除技術因素**，故我把增生 cohort **排除在主要方法比較之外**，未來以 ComBat/RUV 分離。
- ⚠️ 別說「這組特徵對早期訊號敏感」——太滿。

### 15｜HECS 外部 AUC 只比別人高一點，有意義嗎？
外部 AUC 各法本就接近（任務近飽和）。真正差別在**穩定性**：Φ̂ 0.656→0.814，且 HECS 是**唯一收斂成單一可重現共識集合**的方法。定位是穩定性與可重現性，非分類分數。

### 16｜為什麼 HECS 的 Jaccard 算法跟前三法不一樣？
前三法每個種子產生一組特徵 → 算**跨種子**相似度；HECS 收斂成**單一共識集合**、沒有多組可比 → 改用**輪次間**相似度對應。而且**主要**穩定性度量是 **Nogueira**（對集合大小穩健、隨機重疊校正），四法排序一致，Jaccard 只是輔助。

### 17｜為什麼有些 cohort 只報敏感度或只報特異度？
那些 cohort **只有單一類別**，且正常與癌症**來自不同來源**。合併算 AUC 會混入來源差異，不能解讀成區辨能力 → 只報單側率。

# 四、圖表解說（委員指著圖問時）

### 跨種子穩健性熱圖（Shared-Row RRA Input-List Panels）
- 5 個 panel＝5 個隨機種子輪次；**每列＝同一特徵、五圖對齊**，每行＝該輪次一個抽樣集合；顏色越深藍＝該集合排名越前，灰＝沒選到。
- **看點**：同一批列在五個 panel 都呈一致深藍 → 換種子仍選到同一批、排名一樣靠前的特徵。是左邊 0.814／0.690／0.793 的**視覺版**。
- ⚠️ 這畫的是 RRA 的**輸入名單、不是輸出** → 證明穩定性存在於原始排序證據裡，非聚合算出來。列的選取用最終階層 RRA 前 25 名當共同基準。

### 顯著性全景圖 / volcano（左圖，最終共識頁）
- 每點＝一個候選 CpG（共 8,031）；橫軸＝階層 RRA 排名(log)、縱軸＝顯著性(−log10 q)；**紅點＝21 位點**。
- **看點**：紅點全擠在**最左上角**（排名最前＋最顯著），與後面成千候選明顯拉開 → 21 位點不是勉強切出，是最頂層。

# 五、術語紀律 & 提醒

- 軌跡分析只講「集合數累積」；跨種子只講「隨機種子穩健性」——**不混用**。
- 生物一律用「**候選標記／可重現的表觀訊號**」，**絕不用** driver／因果／已驗證標記。
- 少說「臨床區辨價值」→ 改「具穩定下游可用性的候選 CpG 集合」。
- 講「收斂」時心裡清楚：指**核心特徵與排序**，不是集合大小。
- 腫瘤樣本數是 **436**（別口誤講 439）。
